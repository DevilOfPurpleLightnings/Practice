Багатопоточність

Для вирішення проблеми правильної синхронізації та ефективного розподілу ресурсів я реалізував кілька варіантів алгоритму «Решето Ератосфена». Нижче наведено порівняння класичного підходу та різних методів декомпозиції задач у C#.

Послідовний алгоритм (Еталонна реалізація)

Класичне виконання в одному потоці. Створюється масив, де послідовно викреслюються всі числа, кратні знайденим простим. Цей метод містить мікрооптимізацію (одноразове обчислення Math.Sqrt) і слугує базовою лінією (baseline) для оцінки швидкості паралельних версій.

C# 
public static IEnumerable<int> GetPrimeNumbersModifiedSequentialAlgorithm(int n)
{
    ArgumentOutOfRangeException.ThrowIfNegativeOrZero(n);
    if (n < 2) return Enumerable.Empty<int>();

    bool[] isPrime = new bool[n + 1];
    Array.Fill(isPrime, true);
    isPrime[0] = false;
    isPrime[1] = false;

    int limit = (int)Math.Sqrt(n);

    for (int p = 2; p <= limit; p++)
    {
        if (isPrime[p])
        {
            for (int i = p * p; i <= n; i += p)
                isPrime[i] = false;
        }
    }

    return Enumerable.Range(2, n - 1).Where(i => isPrime[i]);
}

Декомпозиція за даними (TPL)

Розпаралелювання внутрішнього циклу за допомогою Parallel.For. Оскільки динамічно задати крок циклу в TPL неможливо, незалежність ітерацій забезпечується простою перевіркою залишку від ділення (i % p == 0).

C# 
public static IEnumerable<int> GetPrimeNumbersConcurrentDataDecomposition(int n)
{
    int limit = (int)Math.Sqrt(n);

    for (int p = 2; p <= limit; p++)
    {
        if (isPrime[p])
        {
            Parallel.For(p * p, n + 1, i =>
            {
                if (i % p == 0)
                {
                    isPrime[i] = false;
                }
            });
        }
    }

    return Enumerable.Range(2, n - 1).Where(i => isPrime[i]);
}

Декомпозиція за базовими простими числами

Використання Parallel.ForEach, де кожен потік отримує своє базове просте число і викреслює кратні йому. Всі потоки вільно пишуть у спільний масив — це створює "безпечний" стан гонитви (race condition), що дозволяє уникнути ресурсомістких блокувань (lock).

C# 
public static IEnumerable<int> GetPrimeNumbersConcurrentBasicPrimesDecomposition(int n)
{
    int limit = (int)Math.Sqrt(n);
    var basePrimes = GetPrimeNumbersSequentialAlgorithm(limit).ToArray();

    bool[] isPrime = new bool[n + 1];
    Array.Fill(isPrime, true);
    isPrime[0] = false; isPrime[1] = false;

    Parallel.ForEach(basePrimes, p =>
    {
        for (int i = p * p; i <= n; i += p)
        {
            isPrime[i] = false;
        }
    });

    return Enumerable.Range(2, n - 1).Where(i => isPrime[i]);
}

Низькорівневе управління (ThreadPool)

Ручне керування пулом потоків замість високорівневих абстракцій. Кожне завдання відправляється в ThreadPool. Для правильної синхронізації (щоб головний потік не повернув результат завчасно) застосовується примітив синхронізації CountdownEvent.

C# 
public static IEnumerable<int> GetPrimeNumbersConcurrentWithThreadPool(int n)
{
    int limit = (int)Math.Sqrt(n);
    using var countdown = new CountdownEvent(limit - 1);

    for (int p = 2; p <= limit; p++)
    {
        int prime = p;
        ThreadPool.QueueUserWorkItem(_ =>
        {
            try
            {
                if (isPrime[prime])
                {
                    for (int i = prime * prime; i <= n; i += prime)
                        isPrime[i] = false;
                }
            }
            finally
            {
                countdown.Signal(); // Звітуємо про завершення
            }
        });
    }

    countdown.Wait();
    return Enumerable.Range(2, n - 1).Where(i => isPrime[i]);
}
